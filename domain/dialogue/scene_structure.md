# Dialogue Session — File Structure

All files for a dialogue session live under `infrastructure/dialogues/{dialogue_id}/`.

---

## `characters.json`

Participants in this session. Written by the frontend at scene setup, updated after leading character selection.

```json
{
  "charA": {
    "id": "string — character directory name (e.g. main_ange_spec_v2)",
    "name": "string — display name",
    "data_path": "string — relative path to data.json (e.g. infrastructure/characters/main_ange_spec_v2/data.json)"
  },
  "charB": {
    "id": "string",
    "name": "string",
    "data_path": "string"
  },
  "leading": {
    "id": "string — set after user selects leading character",
    "name": "string"
  },
  "replying": {
    "id": "string",
    "name": "string"
  }
}
```

`leading` and `replying` are absent until stage 2 (dialogue initialization).

For raw characters pending repack, `data_path` is absent and `needs_repack: true` is present instead.

---

## `scenario.json`

Generated by `optimize_scenario`. Contains each character's adapted scenario and opening line options.

```json
{
  "dialogue_id": "string — UUID",
  "generated_at": "string — ISO 8601 timestamp",
  "characters": {
    "char_a": { "id": "string", "name": "string" },
    "char_b": { "id": "string", "name": "string" }
  },
  "char_a": {
    "scenario": "string — char_a's typical_scenario adapted for char_b",
    "openings": [
      "string — greeting adapted for char_b",
      "string — alternate_greetings[0] adapted for char_b"
    ]
  },
  "char_b": {
    "scenario": "string — char_b's typical_scenario adapted for char_a",
    "openings": [
      "string — greeting adapted for char_a"
    ]
  }
}
```

Both characters' scenarios and openings are always present regardless of which becomes the leading character — that selection happens in the UI after this file is written.

---

## `full_chat.json`

Complete dialogue history. Grows unboundedly. Never trimmed.

```json
[
  { "speaker": "char_id", "text": "string — reply with \\n\\n paragraph breaks" },
  { "speaker": "char_id", "text": "string" }
]
```

---

## `recent_chat.json`

Sliding context window used by agents. Always trimmed to the last **10 entries** after each append.

Same shape as `full_chat.json`. Agents read this, never `full_chat.json`, to bound context cost.

---

## `memory.json` _(optional)_

General scene memory written by agents when notable facts accumulate. Read by `generate_reply` if present.

No fixed schema — free-form JSON object. Not currently generated by any implemented agent; reserved for future use.

---

## Path conventions

- Character `id` in `speaker` fields is the **directory name** (e.g. `main_ange_spec_v2`), not `meta.id` inside `data.json`.
- `data_path` values are relative to the project root: `infrastructure/characters/{id}/data.json`.
